# electron下使用puppeteer模拟用户操作,数据获取

Puppeteer是一个由Google开发的Node.js库，它提供了一组用于控制和自动化Chrome或Chromium浏览器的API。它可以用于执行各种与浏览器相关的任务，如网页截图、生成PDF、模拟用户交互、爬取网页数据等。Puppeteer通过WebSocket连接与浏览器建立通信，并通过DevTools协议发送命令和接收浏览器的响应。

什么是 Chrome DevTool Protocol (opens new window)

  • CDP 基于 WebSocket，利用 WebSocket 实现与浏览器内核的快速数据通道
  • CDP 分为多个域（DOM，Debugger，Network，Profiler，Console...），每个域中都定义了相关的命令和事件（Commands and Events）
  • 我们可以基于 CDP 封装一些工具对 Chrome 浏览器进行调试及分析，比如我们常用的 “Chrome 开发者工具” 就是基于 CDP 实现的
  • 如果你以 remote-debugging-port 参数启动 Chrome，那么就可以看到所有 Tab 页面的开发者调试前端页面，还会在同一端口上还提供了 http 服务，主要提供以下几个接口：
GET /json/version                     # 获取浏览器的一些元信息
GET /json or /json/list               # 当前浏览器上打开的一些页面信息
GET /json/protocol                    # 获取当前 CDP 的协议信息   
GET /json/new?{url}                   # 开启一共新的 Tab 页面
GET /json/activate/{targetId}         # 激活某个页面成为当前显示的页面
GET /json/close/{targetId}            # 关闭某个页面
GET /devtools/inspector.html          # 打开当前页面的开发者调试工具
WebSocket /devtools/page/{targetId}   # 获取某个页面的 websocket 地址

puppeteer 的 API 仿造了浏览器的结构（透明部分表示这些浏览器结构再 puppeteer 中暂未实现:

  • Browser： 对应一个浏览器实例，一个 Browser 可以包含多个 BrowserContext
  • BrowserContext： 对应浏览器一个上下文会话，就像我们打开一个普通的 Chrome 之后又打开一个隐身模式的浏览器一样，BrowserContext 具有独立的 Session(cookie 和 cache 独立不共享)，一个 BrowserContext 可以包含多个 Page
  • Page：表示一个 Tab 页面，通过 browserContext.newPage()/browser.newPage() 创建，browser.newPage() 创建页面时会使用默认的 BrowserContext，一个 Page 可以包含多个 Frame
  • Frame: 一个框架，每个页面有一个主框架（page.MainFrame()）,也可以多个子框架，主要由 iframe 标签创建产生的
  • ExecutionContext： 是 javascript 的执行环境，每一个 Frame 都一个默认的 javascript 执行环境
  • ElementHandle: 对应 DOM 的一个元素节点，通过该该实例可以实现对元素的点击，填写表单等行为，我们可以通过选择器，xPath 等来获取对应的元素
  • JsHandle：对应 DOM 中的 javascript 对象，ElementHandle 继承于 JsHandle，由于我们无法直接操作 DOM 中对象，所以封装成 JsHandle 来实现相关功能
  • CDPSession：可以直接与原生的 CDP 进行通信，通过 session.send 函数直接发消息，通过 session.on 接收消息，可以实现 Puppeteer API 中没有涉及的功能
  • Coverage：获取 JavaScript 和 CSS 代码覆盖率
  • Tracing：抓取性能数据进行分析
  • Response： 页面收到的响应
  • Request： 页面发出的请求

下面是一个puppeteer-in-electron案例

const { BrowserWindow, app } = require("electron"); // 引入electron中的BrowserWindow和app对象
const pie = require("puppeteer-in-electron"); // 引入puppeteer-in-electron模块
const puppeteer = require("puppeteer-core"); // 引入puppeteer-core模块

const main = async () => {
  await pie.initialize(app); // 初始化puppeteer-in-electron
  const browser = await pie.connect(app, puppeteer); // 连接到electron中的浏览器实例

  const window = new BrowserWindow(); // 创建一个electron窗口
  const url = "https://example.com/"; // 定义要加载的网址
  await window.loadURL(url); // 在窗口中加载指定的网址

  const page = await pie.getPage(browser, window); // 通过浏览器实例和窗口实例获取puppeteer页面实例
  console.log(page.url()); // 打印页面的URL
  window.destroy(); // 销毁窗口
};

main(); // 调用main函数开始程序执行

常用的 puppeteerAPI 示例：

  • p**age.goto(url, options)**: 导航到指定的 URL。
  • p**age.reload(options)**: 刷新页面。
  • p**age.waitFor(selectorOrFunctionOrTimeout, options)**: 等待页面上的元素出现或执行某个函数。
  • p**age.type(selector, text, options)**: 在指定的元素中输入文本。
  • p**age.click(selector, options)**: 点击指定的元素。
  • p**age.evaluate(pageFunction, ...args)**: 在页面上下文中执行一个函数。
  • p**age.screenshot(options)**: 截取页面的屏幕截图。
  • p**age.pdf(options)**: 生成页面的 PDF 文件。

# puppeteer常见问题以及解决方法

1**.alert 函数是浏览器原生提供的JavaScript函数,只能由用户手动关闭，不能通过编程方式来控制**

//重写alert方法防止弹框影响到后续代码执行
  await pupPage.evaluate(() => {
    window.alert = (a) => {
      console.log(a);
    };
  });

3.页面加载缓慢获取不到dom节点

如果页面加载缓慢，可以考虑使用 page.waitForNavigation() 方法等待页面加载完成，或者使用 page.waitForSelector() 方法等待特定元素出现。另外，你还可以设置 page.setDefaultNavigationTimeout() 来增加默认的导航超时时间。

await pupPage.waitForNavigation({ waitUntil: "networkidle0" });//等待网络空闲时触发导航完成
//waitForNavigation()方法默认等待所有网络活动完成，包括网络请求和资源加载
await pupPage.waitFor(500);//延时

4.处理弹窗和对话框：当页面弹出警告框、确认框或其他对话框时，可以使用 page.on('dialog') 事件来捕获并处理它们。通过监听 dialog 事件，你可以在对话框出现时进行相应的操作，例如点击确认或取消按钮。

// 监听 'dialog' 事件，处理对话框
page.on('dialog', async (dialog) => {
  console.log('对话框消息:', dialog.message());
  // 可根据对话框消息的内容进行相应的操作
  if (dialog.message().includes('Are you sure you want to proceed?')) {
    await dialog.accept(); // 确认对话框
  } else {
    await dialog.dismiss(); // 取消对话框
  }
});

5.处理登录过期：监听response请求

pupPage.on("response", async (response) => {
    // 检查响应 URL 是否是你要监听的特定响应
    const request = response.request();
    const resourceType = request.resourceType();
    if (resourceType === "xhr") {
      // console.log("响应接收：", response.status(), response);
      const json = await response.json();
      // console.log(json);
      if (json.code === 401) {
        console.log("登录失效");
        const [loginAgain] = await pupPage.$x(
          `//div[contains(@class, "el-message-box__btns")]/button[contains(span, "
          重新登录
        ")]`
        );
        await loginAgain.click();
        await pupPage.waitForSelector("button[type='button']");
        const [zhanghao] = await pupPage.$x(`//input[@placeholder='账号']`);
        await zhanghao.type("admin", { delay: 100 });
        const [mima] = await pupPage.$x(`//input[@placeholder='密码']`);
        await mima.type("admin123", { delay: 100 });
        const [loginButton] = await pupPage.$x(`//button[@type='button']`);
        await loginButton.click();
        console.log("登录成功");
      }
    }
  });

7**.判断单选选项是否被选中**

const radioElements = await mainFrame.$x(`//td[contains(text(), '菜单')]/following-sibling::td/div//label/preceding-sibling::input`);
let onChecked = false;
for (const radioElement of radioElements) {
  const isChecked = await radioElement.evaluate(
    (element) => element.checked
  );
  if (isChecked) {
    onChecked = true;
    break;
  }
}

# XPath表达式 (opens new window)

  1. 选取元素：
    • 选取所有元素： //*
    • 选取指定元素： //div 或者 //span
  2. 属性选择：
    • 选取具有特定属性的元素： //*[@class='classname']
    • 选取具有特定属性值的元素： //*[@id='elementId']
  3. 文本内容选择：
    • 选取包含指定文本的元素： //*[text()='text']
  4. 相对路径选择：
    • 选取当前元素的直接子元素： ./childElement
    • 选取当前元素的所有子孙元素： .//childElement
  5. 层级关系选择：
    • 选取父元素： ../parentElement
    • 选取前一个同级元素： preceding-sibling::siblingElement
    • 选取后一个同级元素： following-sibling::siblingElement
