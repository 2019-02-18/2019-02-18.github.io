JS继承

# JavaScript继承方式分享

在JavaScript中，继承是实现面向对象编程的重要方式之一。

# 原型继承

原型继承是JavaScript中最常用的继承方式之一。在原型继承中，子类通过继承父类的原型对象来实现继承。

function Parent() {
  this.name = 'parent';
}

Parent.prototype.sayHello = function() {
  console.log('Hello from ' + this.name);
}

function Child() {
  this.name = 'child';
}

Child.prototype = new Parent();

var child = new Child();
child.sayHello(); // 输出：Hello from child

原型继承的优点是简单易懂，容易实现。缺点是当父类的原型对象发生变化时，所有继承该原型的子类都会受到影响。

# 构造函数继承

构造函数继承是通过在子类构造函数中调用父类构造函数来实现继承的方式。在这种方式中，子类的实例将会拥有父类实例的所有属性和方法。

function Parent() {
  this.name = 'parent';
}

function Child() {
  Parent.call(this);
  this.name = 'child';
}

var child = new Child();
console.log(child.name); // 输出：child

构造函数继承的优点是可以实现多重继承，缺点是无法继承父类原型对象上的方法。

# 组合继承

组合继承是将原型继承和构造函数继承结合起来的方式。在这种方式中，子类通过继承父类的原型对象来继承父类的方法，而通过在子类构造函数中调用父类构造函数来继承父类的属性。

function Parent() {
  this.name = 'parent';
}

Parent.prototype.sayHello = function() {
  console.log('Hello from ' + this.name);
}

function Child() {
  Parent.call(this);
  this.name = 'child';
}

Child.prototype = new Parent();
Child.prototype.constructor = Child;

var child = new Child();
child.sayHello(); // 输出：Hello from child
console.log(child.name); // 输出：child

组合继承的优点是可以继承父类的属性和方法，缺点是会调用两次父类的构造函数，导致效率较低。

# ES6的class继承

ES6引入了class关键字，使得JavaScript中的继承更加直观和易于理解。在ES6中，可以通过关键字extends来实现类的继承。

class Parent {
  constructor() {
    this.name = 'parent';
  }
  
  sayHello() {
    console.log('Hello from ' + this.name);
  }
}

class Child extends Parent {
  constructor() {
    super();
    this.name = 'child';
  }
}

let child = new Child();
child.sayHello(); // 输出：Hello from child
console.log(child.name); // 输出：child

ES6的class继承的优点是语法更加直观，易于理解和使用。缺点是需要使用新的语法，可能需要对ES6有一定的了解。

# 总结

JavaScript中有多种继承方式，包括原型继承、构造函数继承、组合继承和ES6的class继承。选择合适的继承方式可以提高代码的可读性和可维护性，同时也可以提高代码的效率和性能。在实际开发中，需要根据具体情况选择合适的继承方式，以达到最佳的开发效果。
