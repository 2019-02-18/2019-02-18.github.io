    事件周期

    # electron常用事件统计

    主进程常用事件可以分为以下几类：

    1. 应用程序生命周期事件：
      • ready: 当 Electron 应用程序初始化完成且准备好显示窗口时触发。
      • window-all-closed: 当所有窗口都关闭后触发，在 macOS 上通常不会退出应用程序。
      • activate: 在 macOS 上，当用户点击应用程序的 Dock 图标并且没有其他窗口打开时触发，用于重新创建新窗口。
    2. 窗口相关事件：
      • closed: 窗口关闭时触发。
      • dom-ready: 窗口加载完毕并且 DOM 可以交互时触发。
      • focus: 窗口获得焦点时触发。
      • blur: 窗口失去焦点时触发。
    3. 渲染进程通信事件：
      • ipcMain.on: 监听渲染进程发送的事件。
      • ipcMain.handle: 处理渲染进程发送的请求。
    4. 全局快捷键事件：
      • globalShortcut.register: 注册全局快捷键。
    5. 系统托盘事件：
      • 系统托盘相关事件，如 tray.on("click")，tray.on("right-click") 等。
    6. 自动更新事件：
      • 自动更新相关事件，如检查更新、下载更新、安装更新等。
    7. 系统对话框事件：
      • dialog.showOpenDialog: 显示打开文件对话框。
      • dialog.showSaveDialog: 显示保存文件对话框。
      • dialog.showMessageBox: 显示消息框。
    8. 其他事件：
      • app.on("quit"): 当应用程序退出时触发。
      • app.on("before-quit"): 当应用程序即将退出时触发。

    # pcMain,ipcRenderer,new BrowserWindow这些分别是啥(electron)

    ipcMain、ipcRenderer、BrowserWindow 是 Electron 框架中的三个重要组件，用于实现主进程和渲染进程之间的通信、创建窗口和管理窗口。

    1. ipcMain：ipcMain 是 Electron 主进程中的模块，负责监听和处理来自渲染进程的事件（消息）。使用 ipcMain，主进程可以接收和处理渲染进程发送过来的消息，并可以向渲染进程发送消息。在主进程代码中，您可以使用 ipcMain.on 来监听渲染进程发送的事件，并使用 event.reply 或 event.sender.send 来向渲染进程发送消息。
    2. ipcRenderer：ipcRenderer 是 Electron 渲染进程中的模块，负责监听和处理来自主进程的事件（消息）。使用 ipcRenderer，渲染进程可以接收和处理主进程发送过来的消息，并可以向主进程发送消息。在渲染进程代码中，您可以使用 ipcRenderer.on 来监听主进程发送的事件，并使用 ipcRenderer.send 或 ipcRenderer.invoke 来向主进程发送消息。
    3. BrowserWindow：BrowserWindow 是 Electron 中用于创建窗口的类。您可以使用 BrowserWindow 来创建和管理应用程序中的窗口。主进程可以通过 new BrowserWindow() 创建窗口，而渲染进程可以在页面中使用 window.open 或其他方法来创建新的窗口。通过 BrowserWindow，您可以控制窗口的显示、加载 URL、执行 JavaScript、与窗口中的页面进行通信等。

    在 Electron 应用程序中，主进程负责管理应用程序的生命周期，处理系统级别的操作和与系统资源进行交互，而渲染进程用于加载和显示页面内容，并与用户进行交互。主进程和渲染进程之间的通信需要通过 ipcMain 和 ipcRenderer 来实现，而窗口的创建和管理则使用 BrowserWindow 类。这三个组件共同协作，构成了 Electron 应用程序的核心功能。

    # ipcMain 方法(electron)

    1. ipcMain.on(channel, listener): 监听从渲染进程发送过来的事件。channel 是事件的名称，listener 是一个回调函数，当收到该事件时会执行这个回调函数。
    2. ipcMain.once(channel, listener): 监听一次性的事件，即只有第一次收到事件时会执行 listener 回调函数。
    3. ipcMain.removeListener(channel, listener): 移除指定的事件监听器。
    4. ipcMain.removeAllListeners([channel]): 移除指定通道的所有事件监听器，如果没有指定 channel，则移除所有事件监听器。
    5. ipcMain.emit(channel, ...args): 在主进程中手动触发一个事件。这个事件将在主进程中执行对应的监听器。
    6. ipcMain.sendTo(webContentsId, channel, ...args): 向指定的 webContents 对象发送事件。webContentsId 是目标渲染进程的标识符，channel 是事件的名称，args 是可选的数据参数。
    7. ipcMain.send(channel, ...args): 向所有渲染进程发送事件，不需要指定特定的 webContentsId。
    8. ipcMain.sendSync(channel, ...args): 向渲染进程同步发送事件，并等待直到渲染进程回复。这个方法会导致主进程阻塞，谨慎使用。
    9. ipcMain.handle(channel, listener): 监听渲染进程发送的异步请求，并返回一个 Promise 或回调函数的结果。

    这些是主进程中常用的 ipcMain 方法，它们允许主进程与渲染进程之间进行双向通信和数据交换。需要注意的是，渲染进程中有对应的 ipcRenderer 模块，用于在渲染进程中处理与主进程之间的通信。

    # 主进程和渲染进程通信

    主进程中的代码：

    const { app, BrowserWindow, ipcMain } = require('electron');
app.on('ready', () => {
  const mainWindow = new BrowserWindow();
  mainWindow.loadFile('index.html');

  // 监听渲染进程发送的事件
  ipcMain.on('message-from-renderer', (event, data) => {
    console.log('接收到渲染进程发送的事件：', data);
    // 在这里执行相应的操作，并向渲染进程返回结果
    event.sender.send('message-to-renderer', 'Hello from main process!');
  });
});

    # 自动更新

    1. 配置自动更新服务器： 首先，您需要搭建一个用于存放应用程序更新文件的服务器。这个服务器需要提供一个用于检查更新的 API，以及存放更新文件的路径。
    2. 设置更新策略： 在应用程序的主进程中，需要设置更新策略和配置自动更新的行为。您可以使用 Electron 提供的 autoUpdater 模块来实现自动更新。
    3. 检查更新： 当应用程序启动时，主进程可以通过调用 autoUpdater.checkForUpdates() 方法来检查是否有新的版本可用。
    4. 处理更新事件： 一旦检查到新的版本可用，autoUpdater 会触发相应的事件，如 update-available 和 update-not-available。您需要在主进程中监听这些事件，并根据情况做出相应的处理。
    5. 下载和安装更新： 如果有新的版本可用，您可以通过 autoUpdater.downloadUpdate() 方法来下载更新文件。下载完成后，autoUpdater 会触发 update-downloaded 事件，表示更新已下载完成。
    6. 安装更新： 在 update-downloaded 事件中，您需要调用 autoUpdater.quitAndInstall() 方法来安装更新。这将会关闭当前应用程序，并安装新的版本。

    以下是一个简单的示例代码，演示了如何在 Electron 应用程序中实现自动更新：

    javascriptCopy code
const { app, autoUpdater } = require('electron');

// 设置自动更新服务器的 URL
const server = 'https://your-update-server.com';
const feedURL = `${server}/update/${process.platform}/${app.getVersion()}`;

// 设置更新策略和配置
autoUpdater.setFeedURL(feedURL);

// 监听更新事件
autoUpdater.on('update-available', () => {
  // 有新版本可用，弹出通知或提示用户更新
  console.log('有新版本可用');
});

autoUpdater.on('update-not-available', () => {
  // 没有新版本可用
  console.log('没有新版本可用');
});

autoUpdater.on('update-downloaded', () => {
  // 更新已下载完成，准备安装
  console.log('更新已下载完成');
  // 安装更新
  autoUpdater.quitAndInstall();
});

// 在应用程序启动时检查更新
app.on('ready', () => {
  autoUpdater.checkForUpdates();
});
